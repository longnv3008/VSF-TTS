# Work Rules

## Muc tieu

Day la bo quy tac giup AI lam nhanh ma khong can quet lai repo qua nhieu.

## Reading Strategy

1. Doc `project-memory.md` truoc.
2. Chi mo them phase lien quan.
3. Chi scan source code khi docs chua du hoac khi can verify implementation.

## Update Strategy

Sau moi thay doi co y nghia, cap nhat:

- `project-memory.md` neu thay doi bo canh du an
- phase lien quan neu thay doi theo khu vuc nghiep vu
- `changelog/README.md` de de handoff

## Coding Assumptions

- Backend uu tien chia theo module
- Frontend uu tien chia theo feature
- Runtime data va source code phai tach ro
- Workflow step names nen on dinh vi dang duoc dung cho tracking job

## Handoff Checklist

Truoc khi ket thuc mot dot lam viec, AI nen de lai:

1. Da thay doi gi
2. Canh bao risk neu co
3. File docs nao da duoc cap nhat
4. Viec tiep theo nen lam
